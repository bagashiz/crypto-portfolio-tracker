<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args (process.argv) → dispatch | Local operator input selecting build vs update |
| index.js → Google Sheets API | batchUpdate requests applied to the live spreadsheet; a misrouted range = data loss |
| `.env` → node runtime | SPREADSHEET_ID supplied via `--env-file=.env` (gitignored) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering / data loss | `--build` overwrites an existing populated sheet | mitigate | D-04: build refuses with an error and exits non-zero if DASHBOARD or DCA_LOG tab exists; never calls spreadsheets.create or addSheet on an existing tab (D-01) |
| T-02-06 | Tampering / data loss | `--update` writes/clears the DCA Log data region | mitigate | D-06: update branch appends only Plan 01 builders that are provably bounded above DATA_START_ROW (proven by Plan 01 unit test); no ad-hoc range writes/clears in index.js |
| T-02-07 | Information Disclosure | SPREADSHEET_ID leaks into git via committed config/scripts | mitigate | D-02: ID supplied only via gitignored `.env` through `--env-file`; package.json scripts reference `.env`, never the literal ID |
| T-02-08 | Denial of Service | unhandled API/auth error aborts opaquely | accept | Errors surfaced with actionable messages; this is an on-demand local CLI run by one operator, no availability SLA |
| T-02-SC | Tampering | npm installs | accept | No new package-manager installs this plan — scripts only reference existing `googleapis`; no install task present |
</threat_model>
