<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local filesystem → service-account JWT | `service-account.key.json` is a long-lived credential read from disk by auth.js |
| `.env` → config.js | SPREADSHEET_ID (personal sheet identifier) crosses from a gitignored file into runtime |
| builder request set → Google Sheets API | request objects define ranges; a wrong DCA Log range = irreversible data loss |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | service-account.key.json committed/logged | mitigate | Key is gitignored (`*.key.json`, `service-account.key.json` — .gitignore lines 26-27); auth.js reads from disk only, never logs key contents; no key value in any error message |
| T-02-02 | Information Disclosure | SPREADSHEET_ID in git history | mitigate | D-02: ID sourced from gitignored `.env` (line 19), never the committed config.js; config.js reads `process.env.SPREADSHEET_ID` only |
| T-02-03 | Tampering / data loss | dcaLogUpdateRequests addresses DCA Log data region | mitigate | D-06: update requests provably bounded to rows above DATA_START_ROW; enforced by the data-region-safety unit test in dcaLogSheet.test.js (Task 3) — not by convention alone |
| T-02-04 | Elevation of Privilege | over-broad OAuth scope | accept | Single scope `auth/spreadsheets` (write structure to a sheet already shared with the SA); no Drive-wide or admin scope requested |
| T-02-SC | Tampering | npm installs | accept | No new package-manager installs this plan — `googleapis` already declared in Phase 1; no install task present |
</threat_model>
