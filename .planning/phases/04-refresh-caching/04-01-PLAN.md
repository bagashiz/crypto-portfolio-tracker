Wire all three as editor-callable globals through the existing reserved entry slots.

Purpose: REFRESH-01..04 — the orchestration seam that makes Phase 3's fail-loud providers safe and turns the static Dashboard into a self-refreshing live view.
Output: `Refresh.ts` (+ tests), `Triggers.ts`, two new OAuth scopes, and three new entry globals.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external API → Apps Script | Untrusted provider responses (Hyperliquid/Jupiter) cross into refreshAll; already parsed/validated by Phase 3 providers (fail-loud) |
| Apps Script → Google Sheet | refreshAll writes computed values into Dashboard cells consumed by Phase 5 formulas |
| editor user → ScriptApp | installTrigger/removeTrigger create/destroy time-driven triggers (resource lifecycle) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | refreshAll write into Qty/Price cells | mitigate | A provider failure NEVER writes a non-number (D-07); assembleRefreshRows only emits live/cache/current-sheet numeric values — unit-tested to never inject "ERR"/"#N/A"/null, preventing #VALUE cascade into Phase 5 formulas (formula-injection-adjacent integrity risk) |
| T-04-02 | Denial of Service | installTrigger duplicate triggers | mitigate | Idempotent install (D-09): remove all existing refreshAll triggers before creating one, so repeated installs cannot stack triggers and exhaust the daily trigger/runtime quota |
| T-04-03 | Elevation of Privilege | new oauthScopes (spreadsheets, script.scriptapp) | accept | Scopes are the minimum needed for SpreadsheetApp writes + ScriptApp trigger management; container-bound personal script, read-only external access, no private keys — capability surface widens only to the bound sheet and this script's own triggers |
| T-04-04 | Information Disclosure | status cells / logs | mitigate | refreshAll writes only prices, quantities, timestamps, and Stale? flags — never wallet addresses or the Jupiter API key (mirrors Diagnostics.ts logging discipline) |
| T-04-SC | Tampering | npm/bun installs | accept | This plan adds NO new packages (Apps Script runtime has no npm; all surfaces are built-in CacheService/SpreadsheetApp/ScriptApp). No package legitimacy gate needed |
</threat_model>
