<objective>
Build the two raw-`UrlFetchApp` provider modules — the core deliverable of this phase. Each returns the D-09 contract `Record<id, {price, qty}>`, hiding ticker→id / mint→id translation inside.

Critical testability move (Nyquist Rule): split each provider into (a) thin fetch wrappers that touch the Apps Script globals `UrlFetchApp`/`PropertiesService` (cannot run under `bun test`), and (b) **pure parse functions** that accept an already-parsed response object and return the map — applying the D-09 assembly, D-10 throw-on-missing-price, and D-13 qty-0-on-absent-balance rules. The pure functions are exported and exercised by `parsers.test.ts` with fixture bodies, so the high-risk traversal + fail-loud logic is verified deterministically without a live API.

Purpose: DATA-01 (HL spot prices), DATA-02 (Jupiter prices), DATA-03 (Solana balances via Jupiter Ultra). HL spot balances ship alongside (CONTEXT phase boundary). Phase 4 consumes these maps.
Output: `HyperliquidApi.ts`, `JupiterApi.ts`, `parsers.test.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external API → provider | Untrusted/hostile response body crosses here; must be validated before indexing |
| Script Property (`JUP_API_KEY`) → `x-api-key` header | Secret in transit; must not leak to logs |
| provider → Phase 4 consumer | D-09 map contract; integrity of price (right instrument) matters |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-05 | Tampering | malformed/hostile HL or Jupiter response | mitigate | `muteHttpExceptions:true` + `getResponseCode() !== 200` throw; pure parsers narrow body shape and throw on malformed structure before indexing (D-13); `Number()` parse of string fields |
| T-03-06 | Tampering (integrity) | wrong instrument priced (perp vs spot) → bad PnL | mitigate | HL prices via `spotMetaAndAssetCtxs` index-aligned `midPx`; never index spot by symbol via `allMids` (Pitfall 1); `UBTC` not perp `BTC` |
| T-03-07 | Information Disclosure | `JUP_API_KEY` leaked to logs | mitigate | Key flows only into the `x-api-key` header; no `Logger.log(apiKey)`; verified by grep gate in Plan 03 |
| T-03-08 | Denial of Service | rate-limit / shared-IP 429 | accept | Keyed `api.jup.ag` (D-06) moves egress off the shared keyless-IP bucket; ~4 calls/refresh (D-08) is far under limits; transient 429 surfaces as a throw and self-heals via Phase 4's catch/last-good |
| T-03-09 | Tampering | zero-balance asset wrongly stales venue | mitigate | D-13: balance absence → qty 0 (not throw); only price-absence and HTTP/parse failures throw |
| T-03-SC | Tampering | npm/bun installs | accept | No packages installed (RESEARCH Package Legitimacy Audit: zero installs); raw `UrlFetchApp` only |
</threat_model>
